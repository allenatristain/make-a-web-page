Lesson 1: The Basics of the Web and HTML

Basic Idea of the Web and How it Works

The main components of the web are computers, web browsers, servers, internet, content, and HTTP. The web is the entire collection of interconnected computers and servers which communicate with each other via a protocol known as HTTP. When an individual user visits a web page, the user’s web browser makes a HTTP request, which is sent over the internet, to the server hosting the appropriate HTML document. The server then sends back the HTML document to the user’s computer, which is then interpreted and displayed by the user’s web browser.

Computers are Stupid and Effective

Computers are able to perform tasks which are encoded via a computer language such as HTML. While able to perform complex tasks more effectively than humans, any little error in the code can have huge problems and prevent the code from performing the desired task. Computers read code literally with no room for interpretation or understanding.

HTML and HTML Elements

This first lesson detailed some of the key points of HTML, otherwise known as HyperText Markup Language. HTML documents comprise the majority of the content on the internet. HTML documents contain text content, which is the content seen by users, and markup, which controls how the text content looks. Each HTML document is comprised of HTML elements. These elements come in different sorts and are determined via tags.

Types of Tags

“B” tag - Makes the text content bold.
“M” tag - Makes the text content initialized.
“Break” tag - Makes an one line break, has no closing tag.
“Paragraph” tag - Creates an "invisible box" around the text content.
“Anchor” tag - Makes a link to another page.
“Img” tag - Tags an image.
"Div" tag" - Makes a box around content which allows for markup of the content.
"Span" tag - Allows for markup of a single line of element.

Element and Tag Lingo

Inline/Block- Elements come in either sort. Block elements make an "invisible box" around text content and can have a determinable height and length.
Void Tag - A tag with no closing tag.

Lesson 2: Creating Structure in a HTML Document

Web Development Languages

There are three languages used for web page development, HTML, Java, and CSS. HTML gives structure and content to a page. CSS, otherwise known as Cascading Style Sheets, gives style to the page elements, such as font size, text color, background, position, border, and etc. Java gives the web page an interactive element. While not obvious at first, every web page on the internet is comprised of boxes. Boxes are defined with the "Div" tag. To add style to each box, give each element a class attribute.

The Structure of Web Pages

To see the structure of a web page, in Google Chrome use "Developer tools." What is shown is the structure of the web page as interpreted by the web browser. Generally speaking, the structure shown is described as a tree structure. This is because HTML elements can have other elements nested inside them, and so on and so forth. When examining a web page's code, all the code within the page can be modeled in a Document Object Model (DOM), which is a tree-like structure of a page's code. A good summary about tree structures can be found here. More information about DOMs can be found here.

Translating a Web Page Design into Code

Since every web page is comprised of boxes, start by Boxify a web page design, going from big boxes to small boxes. Boxes can exist within other boxes. The same web page design can be boxifed in different ways.

Lesson 3: Adding CSS Style to HTML Structure

Importance of Avoiding Repetition

It's a good habit to avoid repetition of code whenever possible. This makes your code easier to read and reduces the chances of errors. As an example, when using lots of repetition of styling code, it not only means a greater chance of error, but makes it more difficult to make markup changes to all desired elements. To make a simple markup change, instead of changing a small amount of code in the CSS file, you would have to change the CSS code in each one of the desired elements. This in turn leads to a greater chance of error if some of the elements are missed or incorrectly modified. Why make your coding experience harder?!?

CSS Styling Lingo

Selector - Term used to select all elements of a particular sort in a HTML document.
Class Selectors - Term used to select only a certain elements of a particular sort in a HTML document.

Adding CSS styling to a HTML Document

There are several ways to add CSS styling code to a structured HTML document:

1) Write the CSS styling code in a "Style" tag nested in the “Head” tag of the HTML document. This is fine for very small projects.

2) Link the HTML document to a separate CSS file. Both files are placed in the same project folder. Helps keep organization for larger projects. A “Head” tag is placed at the top of the HTML document with a "Link" tag inside.

3) Adding CSS styling in line with the HTML code. Very bad idea, leads to lots of repetition of styling code, more chances for errors in code.
